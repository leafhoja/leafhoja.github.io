## ① サイト全体のロック・公開（最も重要）

**これが唯一、全ユーザーに即座に反映される制御手段です。**

### VS Code から操作する（推奨）

1. VS Code で **⌘+Shift+P** を押す
2. 「タスク: タスクの実行」を選択
3. 以下から選ぶ：

| タスク名 | 動作 |
|----------|------|
| 🔒 サイトをロック（全ページ閲覧不可） | 全ユーザーに「閲覧できません」を表示して自動 push |
| 🔓 サイトを公開する | 閲覧可能な状態に戻して自動 push |
| 📋 サイトのロック状態を確認 | 現在の状態を表示するだけ（push なし）|

### ターミナルから操作する

```bash
node toggle-lock.js lock     # ロック
node toggle-lock.js unlock   # 公開
node toggle-lock.js status   # 状態確認
```

### 仕組み

`access-control.js` と `index.html` の先頭にある以下の1行を書き換えて push します：

```javascript
var SITE_LOCKED = false;  // false=公開, true=ロック
```

ファイル自体が変わるため、GitHub Pages を通じて**全ユーザーの画面に即座に反映**されます。

## ② 管理者ページ（admin.html）でできること

URL: `https://leafhoja.github.io/admin.html`

> ⚠️ **重要な制限**  
> 管理者ページの設定は `localStorage`（ブラウザのローカルストレージ）に保存されます。  
> **設定が反映されるのは、自分のPCのそのブラウザだけです。** 他のユーザーには影響しません。  
> 全ユーザーへの制御は「① サイト全体のロック・公開」を使ってください。

### ログイン方法

ホームページ（`index.html`）の最下部にある「管理者ページ」リンクをクリック → パスワード入力。

管理者ページは `admin.html` に直接アクセスしても開けます（URLバーに入力）。

### 設定項目

| 項目 | 説明 |
|------|------|
| サイトを公開する | オフにすると自分のブラウザでサイト全体がロックされる（他ユーザーには無効）|
| ページごとの公開設定 | 各ページを個別にオン/オフ（自分のブラウザのみ有効）|
| 閲覧パスワード | ホームページに入る際にパスワードを求める（一度認証すれば他ページはそのまま）|
| Google Analytics | 閲覧状況の確認（外部リンク）|
| Google AdSense | 広告管理（外部リンク）|

## ③ 閲覧者がホームに来たときの流れ

```
アクセス
  │
  ├─ SITE_LOCKED = true? ──── YES ──→ 「閲覧できません」画面（全員共通）
  │
  ├─ ポリシー未同意?  ──────── YES ──→ 同意画面（白背景）
  │                                        └─ 同意 → 次へ
  │                                        └─ 拒否 → 広告→同意画面に戻る
  ├─ 閲覧PW有効 + 未認証? ──── YES ──→ パスワード入力画面（白背景）
  │                                        └─ 正解 → ホーム表示
  │                                           ※ 一度認証すると他ページも自由に閲覧可
  └─ ホーム表示
```

## ④ レッスンページのアクセス制御

レッスンページにはすべて `access-control.js` が読み込まれており、以下をチェックしています：

1. `SITE_LOCKED = true` → `blocked.html`（全体ロック）へ
2. ポリシー未同意 → `index.html` へ（そこで同意を求める）
3. 閲覧PW有効 + 未認証 → `index.html` へ（そこでPW入力を求める）

## ⑥ 新しいレッスンページを追加するとき

1. 既存のレッスンファイルをコピーして作成
2. `<head>` の先頭に `<script src="access-control.js"></script>` が入っているか確認
3. `index.html` のカードリストにリンクを追加
4. `admin.html` の `PAGES` 配列にエントリを追加（任意：管理画面でページ別制御したい場合）

## ⑦ パスワード関連

| パスワード | 保存場所 | 変更方法 |
|-----------|---------|---------|
| **管理者パスワード** | `index.html` 内にPBKDF2ハッシュとして埋め込み | ソースを直接編集してpush |
| **閲覧パスワード** | 各ユーザーのlocalStorage | admin.htmlの「閲覧パスワード設定」から変更 |

> 管理者パスワードは12文字以上・英数字記号混在を推奨します（ハッシュがソースに公開されているため）。
